Reference deck

Core Web3 ideas, side by side.

Each card compares two related ideas so the differences are easy to scan, remember, and explain.

Web2 vs Web3

Web2

Apps are owned by companies. Your account, data, and permissions usually live on private servers.

Web3

Apps can use public networks. Wallets, tokens, and smart contracts let users carry ownership across apps.

Ethereum vs Bitcoin

Bitcoin

Designed mainly as digital money with a simple, durable rule set for sending and storing value.

Ethereum

Designed as programmable infrastructure where developers deploy smart contracts and decentralized apps.

Public Key vs Private Key

Public Key

Your visible receiving identity. It can be shared so others can send assets or verify signatures.

Private Key

Your secret signing power. Anyone who gets it can control the wallet, so it must stay protected.

Blockchain vs Traditional Databases

Database

A central operator can edit records, control access, and optimize for fast private application data.

Blockchain

A shared network records changes in linked blocks, making history transparent and difficult to rewrite.
